# Book_Management_System
Problem Statement: Book Management System 
Objective Develop a robust Book Management System using Java and Maven that enables an administrator to perform CRUD (Create, Read, Update, Delete) operations on a library catalog. The system must be designed with an emphasis on Object-Oriented Programming (OOP) principles to ensure scalability and maintainability.
Functional Requirements The application must support the following administrative operations: 
Add a New Book: Allow the admin to input and store details for new book entries. 
View All Books: Retrieve and display a comprehensive list of all books currently in the system. 
Update Book Price: Modify the price of an existing book. This operation must require both the Book ID and the Book Author for verification before updating. 
Delete Book: Remove a specific book record from the system using its unique Book ID.
Search by Title: Search for and display full details of a book based on its Title. 
Technical Specifications To ensure high-quality software engineering, the solution must incorporate: 
Runtime Polymorphism: Implement interfaces or abstract classes (e.g., a generic Item class or a BookService interface) to demonstrate dynamic method dispatch. 
Code Readability: Use meaningful naming conventions, proper indentation, and clear documentation. 
Reusability: Organize the project into distinct layers (e.g., Model, Service, and Main) to allow components to be reused in future modules. 
Project Management: Use Maven for handling dependencies and project build lifecycle.
